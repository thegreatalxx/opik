Opik Connect is a CLI command that bridges your locally running agent to the Opik platform. It allows Opik to send execution requests to your agent and receive results back, enabling the [Agent Sandbox](/agent-development/agent-sandbox) and [Ollie](/agent-development/ollie) workflows.

## How it works

Opik Connect acts as a thin wrapper around your normal startup command. It:

1. Exchanges a pairing code for runner credentials
2. Injects environment variables into your process
3. Launches your command as usual

Your agent code runs exactly as it would without Opik Connect — the only difference is that Opik can now dispatch jobs to it.

## Getting started

### 1. Mark your agent as an entrypoint

Add `entrypoint=True` to your `@opik.track` decorator:

```python
import opik

@opik.track(entrypoint=True)
def my_agent(user_message: str) -> str:
    """Process a user message and return a response."""
    # Your agent logic here
    return response
```

The function's name becomes the agent name in Opik. Its parameters define the input form in the Agent Sandbox, and the docstring is shown as the agent description.

### 2. Get a pairing code

Navigate to your project in the Opik UI and open **Agent sandbox** from the sidebar. The connection screen displays a pairing command with a generated code:

```bash
opik connect --pair abc123 -- <your app start command>
```

### 3. Run the connect command

Replace `<your app start command>` with whatever you normally use to start your agent:

```bash
# Simple script
opik connect --pair abc123 -- python my_agent.py

# Web server
opik connect --pair abc123 -- uvicorn app:main --port 8000

# Any executable
opik connect --pair abc123 -- node agent.js
```

Once connected, you'll see a confirmation message:

```
Runner connected (ID: a1b2c3d4-...).
```

<Note>
The `--` separator is required between the Opik Connect options and your command.
</Note>

## CLI reference

```
Usage: opik connect [OPTIONS] -- COMMAND [ARGS]...

Options:
  --pair TEXT   Pairing code from the Agent Sandbox UI
  --name TEXT   Custom runner name (defaults to hostname + random suffix)
```

### Options

| Option | Description | Required |
|--------|-------------|----------|
| `--pair` | The pairing code generated by the Agent Sandbox UI | Yes |
| `--name` | A custom name for this runner instance | No |

## Defining agent parameters

The parameters of your entrypoint function automatically become input fields in the Agent Sandbox. Opik uses type annotations to determine the field types:

```python
@opik.track(entrypoint=True)
def my_agent(
    user_message: str,
    temperature: float,
    max_tokens: int,
    verbose: bool,
) -> str:
    """An agent with typed parameters."""
    ...
```

This creates four input fields in the sandbox UI with appropriate controls for each type.

Supported parameter types:

| Python type | Sandbox UI control |
|-------------|-------------------|
| `str` | Text input |
| `int` | Number input (integer) |
| `float` | Number input (decimal) |
| `bool` | Toggle switch |

## Compatibility

Opik Connect works with any process that stays alive after startup. This includes:

- Python scripts with blocking calls (e.g., `input()`, event loops)
- Web servers (Flask, FastAPI/uvicorn, Django)
- Long-running services
- Any framework that keeps the process running

<Warning>
If your process exits immediately after loading (e.g., a script that finishes without blocking), the runner won't be able to receive jobs. Use a framework like Flask or uvicorn, or add a blocking call to keep the process alive.
</Warning>

## Environment variables

When Opik Connect launches your command, it sets the following environment variables:

| Variable | Description |
|----------|-------------|
| `OPIK_RUNNER_MODE` | Set to `"true"` to activate the runner loop |
| `OPIK_RUNNER_ID` | The unique ID assigned to this runner |
| `OPIK_PROJECT_NAME` | The project name this runner is associated with |

These are used internally by the Opik SDK. You should not set them manually — always use `opik connect` to launch your process.

## Troubleshooting

### Runner disconnects after starting

Make sure your process stays alive after loading. If you see a warning about the process exiting without blocking, your script is finishing before the runner can process any jobs.

### "Command not found" error

Opik Connect resolves the command against your `PATH`. Make sure the command is installed and accessible in your terminal.

### Pairing code expired

Pair codes are short-lived. If your code has expired, go back to the Agent Sandbox UI — a new code is generated automatically.
